ERROR 2243
Creality Ender 3 Slicing Error

Creality Ender 3 Error 2243 — Coordinate Exception Fix Guide

Error 2243 on the Creality Ender 3 indicates a coordinate exception, almost always caused by mismatched settings between your slicer software and the printer firmware. This guide covers slicing software diagnosis, wrong model selection fixes, third-party G-code troubleshooting, recommended slicer settings, SD card issues, and firmware versus slicer compatibility for all Ender 3 variants including the Ender 3, Ender 3 Pro, Ender 3 V2, and Ender 3 S1.

What Usually Goes Wrong?

Error 2243 on the Ender 3 is a coordinate exception triggered when the printer receives movement commands that fall outside its physical build volume or conflict with its defined coordinate system. This is almost never a hardware fault. The root cause is almost always in the slicer settings or the G-code being sent to the printer. Understanding what each slicer setting does and how it maps to the Ender 3 firmware will resolve the vast majority of Error 2243 cases.

  • • Wrong printer model selected in slicer: the most common cause. Selecting a larger or smaller printer profile produces coordinates that exceed Ender 3's 220 x 220 x 250mm bed.
  • • Third-party G-code not generated for Ender 3: pre-downloaded prints or G-code from other printers may have absolute positioning or bed sizes incompatible with Creality machines.
  • • Incorrect home offset or bed leveling values: custom firmware with modified home offsets can push coordinates out of range.
  • • SD card file corruption or formatting issues: a damaged or incorrectly formatted SD card can cause file read errors that manifest as coordinate exceptions.

Work through these steps in order. Most Error 2243 issues are resolved by simply regenerating the G-code with the correct Ender 3 printer profile selected.

Quick Symptom Check

Answer these before proceeding:

  • Did you recently change your slicer software or update it?
  • Did you download this G-code file from somewhere else?
  • Does the error occur immediately when the print starts?
  • Did you change any firmware settings recently?
  • What slicer are you using: Cura, PrusaSlicer, Simplify3D, or other?

Tools & Materials You'll Need

Software

  • Cura, PrusaSlicer, or other slicer installed
  • Access to printer manufacturer's firmware notes
  • SD card formatter (SD Memory Card Formatter by SD Association)
  • Text editor for G-code inspection (VS Code, Notepad++)

SD Card

  • Known-good SD card (8GB or 16GB recommended)
  • FAT32 formatted (required for Ender 3)
  • SD card reader for your computer
  • Backup and re-format if corruption suspected

Printer Settings

  • Ender 3 build volume: 220 x 220 x 250mm
  • Origin at bed center or front-left depending on firmware
  • Nozzle diameter: 0.4mm (standard)
  • Max print speed reference from manufacturer

Diagnostic Tree — Work These In Order

01

Verify the printer model selected in your slicer

The single most common cause of Error 2243 is selecting the wrong printer model in your slicer software. If you accidentally selected a printer with a larger build volume (such as an Ender 5 with 300x300 bed or a CR-10 with 300x300x400), the slicer will generate G-code with coordinates that fall outside the Ender 3's 220 x 220 x 250mm boundaries.

Correct Ender 3 Printer Profile Settings
Build volume: 220 x 220 x 250 mm
G-code flavor: Marlin (or Creality S3R for S1)
Nozzle diameter: 0.4 mm
Start G-code: Includes G28 homing
End G-code: Includes G28 or G29 if enabled
Positioning: Relative or Absolute per firmware
02

Check the G-code file origin and regeneration test

If the print file was downloaded from Thingiverse, Printables, Cults3D, or any other third-party site, the G-code was likely generated for a different printer. Even if the model looks like it would fit on an Ender 3, the coordinates and start/end sequences may be incompatible. Always re-slice downloaded STL files using your own slicer with the correct Ender 3 profile.

G-code inspection: Open the .gcode file in a text editor and look at the first 20 lines. Check for G0 and G1 movement commands with X coordinates greater than 220, Y coordinates greater than 220, or Z coordinates greater than 250. Also check if the start code references a different bed size in comments or parameters.

03

Inspect G-code positioning mode (absolute vs relative)

Different slicers and firmware versions handle positioning differently. Creality Ender 3 firmware typically uses absolute positioning by default, but some slicers generate relative positioning (G91 mode). If your G-code mixes positioning modes incorrectly or the printer receives commands in the wrong mode, coordinate exceptions can occur. Check your start G-code for G90 (absolute) or G91 (relative) and ensure it matches your slicer output.

G-Code Positioning Reference
  • G90: Absolute positioning — all coordinates are relative to origin (home position)
  • G91: Relative positioning — each move is relative to current position
  • Marlin default: G90 (absolute) on Ender 3
  • Mix-up symptom: Print head moves in wrong direction or distance during first layer
04

Reformat SD card and test with known-good file

SD card corruption can cause files to be read incorrectly, resulting in malformed G-code that triggers coordinate exceptions. Use the official SD Memory Card Formatter tool (available from sdcard.org) to format the card with FAT32 filesystem. After formatting, copy only a single known-good G-code file to the card and test if the error persists. Avoid using large capacity cards (64GB+) as some Ender 3 models have limited compatibility.

SD card compatibility: Ender 3 standard firmware supports SD cards up to 32GB formatted as FAT32. Cards formatted as exFAT or NTFS will not work. Ender 3 S1 Plus may have different limits — check your specific model's documentation.

05

Check slicer bed origin and home offset settings

The Ender 3 homes to the front-left corner by default (MIN endstops). If your slicer is configured for center-origin bed positioning or has custom home offset values (M206 command), the sliced coordinates may not align with the printer's actual home position. Verify that your slicer's "Build plate origin" matches your firmware home position. Some slicers allow origin at center of bed — this can cause coordinate shifts on Ender 3.

06

Review third-party slicer compatibility settings

If you are using Simplify3D, Kisslicer, IdeaMaker, or any slicer other than Cura or PrusaSlicer, the printer profile definitions may be outdated or incorrect. These slicers maintain their own printer definition databases and may not have the correct Ender 3 bed size, filament diameter, or G-code flavor settings. Check the slicer's configuration wizard or machine settings panel and ensure all dimensions and G-code flavor settings are correct.

Recommended Ender 3 Slicer Settings
  • Filament diameter: 1.75mm (measure your actual filament with calipers)
  • Nozzle diameter: 0.4mm default (0.2mm or 0.8mm if changed)
  • G-code flavor: Marlin for standard Ender 3, Creality S3R for Ender 3 S1
  • Start G-code: Must include G28 (home all axes) before print moves begin
  • Layer height: 0.12mm to 0.28mm typical for 0.4mm nozzle
07

Verify firmware vs slicer compatibility and restore defaults

If you have recently flashed custom firmware (Marlin, Klipper, or TH3D Unified Firmware), the coordinate system behavior may have changed. Some custom firmware configurations change the home offset behavior, enable software endstops differently, or alter the bed size definitions. If you suspect firmware changes caused Error 2243, reflash the firmware or compare your Configuration.h settings against the known-good defaults for your Ender 3 model.

  • Marlin Configuration.h: Check X_BED_SIZE, Y_BED_SIZE, Z_MAX_POS match 220, 220, 250.
  • Software endstops: Custom firmware may have disabled or changed software endstop behavior. Verify MIN_SOFTWARE_ENDSTOPS or MAX_SOFTWARE_ENDSTOPS settings.
  • Restore firmware: If all else fails, reflash the factory Creality firmware or reinstall the known-good slicer profile and regenerate the G-code from scratch.

Firmware-Specific Notes

Creality Stock Firmware

Creality's stock firmware for Ender 3 uses absolute positioning with G90 as default. The bed size is defined as 220x220 and the printer will reject any G-code with coordinates outside this range. If you receive Error 2243 on stock firmware, the G-code is almost certainly generated for a different printer or the slicer printer profile is wrong. Re-slice with the correct Ender 3 profile.

Marlin Firmware

Marlin firmware on Ender 3 allows more configuration flexibility but also requires correct Configuration.h bed size definitions. If you have compiled your own Marlin build, verify the bed dimensions match your physical printer. Marlin also supports software endstops that will prevent moves beyond defined boundaries — this is a safety feature that can trigger coordinate errors if misconfigured.

Klipper on Ender 3

Running Klipper on Ender 3 requires correct [probe] and [stepper_x/y/z] definitions in the printer.cfg file. The max_velocity and max_z_velocity settings must match your printer's mechanical limits. If you are getting coordinate exceptions after switching to Klipper, verify the min/max position values in your Klipper config match 0-220 for X, 0-220 for Y, and 0-250 for Z.

TH3D / Other Custom Firmware

Third-party firmware like TH3D Unified Firmware includes pre-configured profiles for Ender 3 variants. These firmware builds often include auto-leveling compensation and custom home offsets. If Error 2243 appeared after flashing a custom firmware, check the firmware's documentation for any changed default behavior related to coordinate system or bed size definitions. Some TH3D builds include G29 auto-leveling in the start code — this requires the probe to deploy properly.

When to Call a Professional or Replace Parts

Error 2243 occurs on EVERY G-code file including files you just re-sliced with correct settings — firmware corruption likely, reflash firmware.

Printer moves to wrong coordinates even with freshly sliced G-code — stepper driver or mainboard failure may be misdiagnosed as coordinate error.

SD card slot is physically damaged or SD card not detected at all — mainboard SD card reader replacement needed.

If the printer is under warranty and you suspect firmware or mainboard issues, contact Creality support before attempting firmware reflash.
